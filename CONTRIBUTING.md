# Contributing to Kable

Thanks for taking an interest in Kable.

Kable is early in development, so small, focused contributions are the
easiest to review and build on.

## Project Direction

Kable is a Python terminal IDE built around Textual. The project code lives in `/main`

## Ways to Contribute

- Report bugs with clear reproduction steps.
- Suggest IDE features or workflow improvements.
- Improve documentation.
- Help shape the initial terminal UI application structure.
- Add focused implementation changes once the project has a runnable
  scaffold.

## Development Guidelines

- Keep changes small and focused.
- Follow the style of the surrounding code and documentation.
- Prefer clear names over clever abstractions.
- Update documentation when behavior, setup, or project structure
  changes.
- Add tests when there is test infrastructure and the change has
  behavior worth protecting.

## Commit Messages

Use short, descriptive commit messages. A good message explains what
changed in plain language.

Examples:

```text
Added initial terminal app scaffold
Document development setup
Fixed editor tab close behavior
```

## Pull Requests

Before opening a pull request:

- Make sure your branch is up to date.
- Describe what changed and why.
- Mention any manual testing you performed.
- Link related issues when applicable.

## Issues

For bug reports, include:

- What you expected to happen.
- What actually happened.
- Steps to reproduce the issue.
- Your operating system and Python version, if relevant.

For feature requests, include:

- The workflow or problem the feature supports.
- Any relevant examples from other IDEs.
- Whether the feature is essential for an early version or can wait.

## Code of Conduct

Be respectful, constructive, and patient. Kable should be a good place
to learn, build, and collaborate.
